# TOP Dashboard Project

## Traduzione Italiana delle Istruzioni

Ora che hai avuto molta pratica usando Grid, costruiremo un design completo di dashboard. Per questo progetto, usa qualsiasi strumento di cui hai bisogno per portarlo a termine, ma cerca di affidarti a Grid per la maggior parte del lavoro di layout. Torna indietro attraverso le lezioni o gli esercizi di pratica se hai bisogno di un ripasso.

### Progetto Dashboard

#### Compito
**Passaggio 1: Impostazione e pianificazione**
Imposta il tuo repository Git (fai riferimento a progetti passati se hai bisogno di un ripasso).
Imposta i tuoi file HTML e CSS con alcuni contenuti fittizi, solo per assicurarti di avere tutto collegato correttamente.
Scarica una copia a piena risoluzione del file di design del progetto e prendi un'idea generale di come dovrai disporre le cose nel tuo documento HTML.

**Passaggio 2: Layout**
Inizia scrivendo gli elementi HTML per i contenitori sidebar, header e main-content.
Nel tuo file CSS, applica le proprietà Grid fino a quando hai costruito questo layout di base.

**Passaggio 3: Annidamento**
Prendendo una sezione alla volta, inizia ad annidare gli elementi figlio sotto gli elementi genitore nell'HTML. Ricorda che puoi continuare a creare contenitori grid all'interno di contenitori grid.
Nella sidebar, usa più grid per disporre le sezioni di navigazione e branding.
Nell'header, usa più grid per disporre la barra di ricerca, le informazioni utente e i pulsanti.
Per il main-content, usa più grid per disporre i progetti, gli annunci e gli elementi di tendenza.
Compila alcuni contenuti fittizi e immagini segnaposto in modo da poter posizionare tutti i tuoi elementi grid.

**Passaggio 4: Raccogli assets**
Una volta completato il tuo grid layout, puoi ricreare l'esempio di dashboard sopra o stilizzare il tuo design.
Tutte le icone e altro possono essere scaricate come SVG da Material Design Icons.
Scegli i tuoi font! L'esempio di design usa Roboto, che è disponibile con Google fonts. (Potresti voler rivedere la Lezione More Text Styles per informazioni su come includere font esterni nei tuoi progetti).

**Passaggio 5: Alcuni suggerimenti!**
Quando costruisci il layout, applica colori di sfondo o bordi ai tuoi contenitori per aiutarti a visualizzare la tua grid.
Sta a te decidere se usare pixel, unità fr o entrambi per le tue tracce grid.
Questo progetto non deve essere responsive, ma se vuoi puoi espandere o restringere la sezione delle schede progetto quando ridimensioni la finestra del browser.
Non devi fare una corrispondenza perfetta in pixel con l'esempio di design. Considera questa un'opportunità per praticare le tue abilità CSS con i tuoi design.
Non dimenticare di pushare la tua dashboard finita su GitHub. Usa GitHub Pages per pubblicarla al mondo!

**Passaggio 6: Feedback della sezione**
Prima di procedere, ci farebbe piacere se potessi inviare il tuo feedback sul corso Intermediate HTML and CSS.

Now that you've had plenty of practice using Grid, we are going to build a full dashboard design. Go back through the lessons or practice assignments if you need a refresher.

### Dashboard project

#### Assignment
**Step 1: Set up and planning**
Set up your Git repository (refer to past projects if you need a refresher).
Set up your HTML and CSS files with some dummy content, just to make sure you have everything linked correctly.
Download a full-resolution copy of the project design file and get a general idea for how you're going to need to lay things out in your HTML document.

**Step 2: Layout**
Start by writing out the HTML elements for the sidebar, header and main-content containers.
In your CSS file, apply Grid properties until you have this basic layout built.

**Step 3: Nesting**
Taking it one section at a time, begin nesting child elements under the parent elements in the HTML. Remember that you can keep making grid containers within grid containers.
In the sidebar, use more grids to lay out the navigation and branding sections.
In the header, use more grids to lay out the search bar, user info and buttons.
For the main-content, use more grids to lay out the projects, announcements and trending items.
Fill out some dummy content and placeholder images so you can position all of your grid items.

**Step 4: Gather assets**
Once you have your grid layout complete you can either recreate the dashboard example above or style your own design.
All of the icons and more can be downloaded as SVGs from Material Design Icons.
Choose your own fonts! The design example uses Roboto, which is available with Google fonts. (You may want to review the More Text Styles Lesson for information about how to include external fonts in your projects).

**Step 5: Some tips!**
When building the layout, apply background colors or borders to your containers to help you visualize your grid.
It's up to you whether to use pixels, fr units or both for your grid tracks.
This project does not have to be responsive, but if you'd like to you can expand or shrink the project cards section when resizing the browser window.
You don't have to make a pixel perfect match with the design example. Consider this an opportunity to practice your CSS skills with your own designs.
Don't forget to push your finished dashboard to GitHub. Use GitHub Pages to publish it to the world!

**Step 6: Section feedback**
Before you move on, we would love it if you could submit your feedback on the Intermediate HTML and CSS course.
